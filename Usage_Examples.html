How to use this program...

Basic operations

Start the application and click onto the "open" button to load an Image.


Click onto the "Go" button...


...to get to this window:


Make sure, the "rect Mode" checkbox is checked and draw a rectangle around the desired object with the mouse, left button pressed.


Click onto the "Go" button...


...to get a result. As you see, the result is not optimal, with some parts being classified as background, which definitely are foreground...


...usually, you now would mark these parts in the results pane as foreground, by checking the "draw on result" checkbox, selecting "foreground" in the dropdown combobox and drawing onto the result. Then clicking "Go" again. But we now will click the "reset"-button, ...


...re-check the "rectMode" checkbox, draw a new Rectangle, maybe somewhat tighter around the objects boundaries, and you could click onto "Go" again. ...


... But because here we will test, if a closed-form-matte computation with some predefined parameters will resolve the issues, we "Reset" and simply click onto "DoAll".


This will lead us to a matte that shows, that its not good enough for production use, but ok for the demo right now.


Now we will test this picture in a new background, and so, we click the "compose"-button.


...which will lead us to the following view.


Clicking onto the "Go" button (which will be renamed into something related to background), and selecting an image shows up something like this...


...or this. Move around the object with your mouse (left button pressed). Rotate it with the right mouse-button pressed.


 

Now, we will show a different use, that may be sufficent for pictures that are hard to segment with the previous method. Again, we load an image, but now we check the "scribbleMode" checkbox and check the "Unknown" radio button. Now we could draw an unknown region in the picture with the mouse. But we want to make it somewhat easier for us, an so we also check the "clickMode" checkbox...


... and click repeatedly onto the boundary of our desired object, beginning at a random point on the border of the object, then following the boundary until the "unknown-curve" is closed...


Now click - with the right mouse button - into the part, thats enclosed by the curve and mark it as foreground, by clcking the "FloodFG" context menu label...


... leading to a white overlayed foreground part. A click onto the "Compute a matte" button ...


... will show up this form - in which all the drawing also could have been done. If the drawings seem alright, press the "gen trimap" button, to create a grayscale picture with the foreground parts as white, the background parts as black, and the unknown part in a gray color.


The trimap bitmap will be shown zoomed in the picturebox. Now click onto the "Go" button, to start the computation...


... with a result like this (After closing the window, which shows the alpha-matte).


The result shows a part (near the boundary left of the nose, top of the right arm of the teddy), where the picture is not as transparent, as needed...We will solve this by drawing the affected region as background. To do so, click the "BG" radiobutton and select a reasonable size (the numeric control left of the "BG" radiobutton, labeled "width") for the strokes being drawn with the mouse. To have a better view, you could check the "overlay img" checkbox, which should do as expected. I also redrawed a part of the foreground ("FG" radiobutton) on the left arm of the teddybear.


...re-generate the trimap...


...and click "Go" again...


... to get an acceptable result


Since we computed the matte in "reduced size mode" (see the checkbox), the gradient of the alpha channel might not be sufficient. We could change this by setting a range from 0 to the desired value to zero in the alpha channel - and recompute the values up to the "fully opaque level", which is 255 in this case. Also it is possible to change the alpha gamma.

Pseudo shadow

Load the extracted object into the "compose"-form, select a background and click on the "CloneUpperShape" - button.


Select an opaque region of the newly presented shape and click the "Floodfill" - button, to make all content of this shape black.


Now click the "Gaussian-blur" - button and ...


... select a reasonable value to get the shape blurred.


Shear the shape to fit the lighting in the background picture. (Please note that the undo operation for this will be revisited, to set again the size of the unsheared image).


Now uncheck the "keep aspect" checkbox...


...and set the width and height according to your composition imagination. Move the shape to its final position to test the size.


Decrease the opacity...


... and click onto the original shape, to bring it to front.


"Merge" the scene into one layer (button "merge"), or simply...


... "OK" out of the form. The scene will be auto-merged then.

If a small window will be opened, dont care, just close it, its just a debugging helper image that displayes the current mask being used in the computation...

Transparent inner parts, number of components

Sometimes your picture will contain inner parts that should be treated as background. The normal result looks like this...


... but you can increase the amount of initially displayed components by setting the desired value in the "max # components" numeric-control.


 

Automatic threshold evaluation vs. setting the parameters manually

Sometimes the automatic evaluation of the threshold for segmenting produces not very good results...


... if you uncheck the "assume exp dist in dH" checkbox, it might get better...


... or simply uncheck the "auto threshold" checkbox too and set the parameters manually. A value of 13.5 in this case is already a bit too much. Increasing the threshold value will lead to more pixels being classified as background. Play around with the parameters a bit to get some feeling for it.


 

A scenario when the GrabCutALike algorithm doesn't produce the desired results and just creating a Matte is not enough, because the hard extracted feature picture has to be saved

Consider a Photography where this algorithm doesnt work well


Click onto the preBlur button


First we stretch the lower colors a bit and ok out of this form


Then we click the outline button


In this form we repeatedly click near the boundary of the object to extract, and so, surround the object once


If the underlying algorithm dont follow the outline correctly, you could change the parameters, or, simply, click the remSeg button to remove the last segment, check the addLine checkbox and simply add some lines by clicking. When you think the algorithm could take over again, uncheck the addLine checkbox and proceed clicking the boundary


When you are close to closing the path, click the closePath button


Now click the crop button


You could add a set of unknown scribbles to the picture in the left pane of the main extracting form, but we here want to get the extracted object in the right pane of the main extracting form. So we simply click ok button, since the correct option is preselected


In this form we now will process the outline of the extracted object


In the process outline form, click the res vals button and increase the inner processing range a bit. Then check the edit trimap checkbox and click onto create Trimap


This will show the auto created trimap. Check the overlay orig image checkbox and ...


... draw the whiskers part as unknown. Remeber: White is foreground, black background and the unknown parts are coded in gray. Ok out of this form


Now uncheck the edit trimap button and click onto Go


After computing the matte, the program will show it to you. Maybe save it now and click close


Inspecting the result will show some problems: By using the reduced size option in matte creating, the whole alpha ranges are a bit too wide. See the back of the cat. Some false portions are computed as partly foreground and the whiskers are not extracted correctly. The last appears because of the photo's composition. Usually the closed form matte is very good at extracting wispy parts like hairs and so on, but here the whiskers are not always drawn really continuous and the background is changing a lot. So the whiskers are sometimes lighter than the BG but also sometimes darker. This makes it impossible for the algorithm to extract these parts correctly


So we ok out of the form and in the main extracting form we click the OutlineOPs button


Here we check the wipeAlpha checkbox and chose a somewhat bigger pen. Then we wipe the incorrectly extracted parts


Keep the pen's Alpha value at around 0.7 to do so softly


When done, ok out of the form and in the main extracing form click onto the compose button. Load the desired background and look at the picture to see that the whiskers are really missing in this composition


We now can solve the wide alpha range problem by changing the gamma (power function) of the alpha channel


Return to the main extracting form and click onto PoissonDraw. We now will use Poisson blending for pen strokes to add the whiskers to the picture. This is a technique of derivative based image editing. I.e.: We want to use the colors of the lower image at the boundary of our pen strokes, but the Change (= structure) of the newly drawn parts (usually of the upper image in Poisson blending)


In the Poisson drawing form first load the original image to the left pane from which we will copy the whiskers


Now define a src point in the left pane and a dest point in the right pane. These points are mapped/synchronized to virtually overlay the left picture onto the right one at the correct coordinate. Use points that are easy to find in both images, to get enough of precision for overlaying the images


You can check, if your points are good enough by switching on and off the overlay src pic checkbox.


Now check the Draw checkbox, chose a relatively small pen and follow the whiskers with the left mouse button pressed in the right pane. Do one whisker at a time for keeping the copied parts slim, to not copy too much of the undesired background. Check the result by unchecking the overlay src pic checkbox


Do so for the ones over the cats eye


Ok back to the main extracting form. Save the picture


Here's a screenshot of the result


Some notes if this procedure produces too much src pic background in the result. Since we are using a changing and colorly close src pic, poisson blending with the unedited src will produce nice results. This usually is not the case. For this you can expose the lighter parts of the src image by screen blending it to itself, then do the poisson drawn pen strokes. Then go back to the main extracting form and chose compose again. Now click onto the From cache button and select the previously composed picture (cat at correct place but missing whiskers) as new background. Now you have the same scene twice, one (BG) without the whiskers and one (FG) with the Poisson blended result. Click onto the foreground picture and reduce the opacity of it until you like the results. Pictures for this procedure will follow
